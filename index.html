Version controller System

A Version Control System (VCS) is a tool that helps track changes in code, documents, and files over time, allowing collaboration and rollback to previous versions when needed.

Types of Version Controller System
  1. Local Version Control
    • Simple database stores file versions locally.
    • Example: Backup copies with timestamps.
  2. Centralized Version Control (CVCS)
    • A single central server holds all versions.
    • Developers check out and commit changes to this server.
  3. Distributed Version Control (DVCS) (Most Popular)
    • Every developer has a full copy of the repository.
    • Allows working offline and merging branches easily.
